# Admin Console Rules

Read-only internal admin surface at `/admin`. Distinct from `(app)`: separate route group, separate layout, separate auth gate (`requireAdmin` instead of `getCurrentUser`).

- **`requireAdmin()` in `lib/auth/admin.ts` is the only sanctioned admin gate.** Server Components and admin route handlers MUST call it first thing. On non-admin or unauth: `notFound()` — never `redirect("/sign-in")`. 404-not-403 makes the surface invisible to enumeration. The `(admin)` layout calls it once so child pages inherit.
- **`ADMIN_EMAILS` env var is comma-separated allowlist.** Empty/unset env defaults to LOCKDOWN (no admins) so misconfigured prod doesn't accidentally grant admin to everyone. Bind via `firebase apphosting:secrets:set ADMIN_EMAILS`. `.env.local` for local dev only.
- **Admin reads go through `lib/firebase/admin-queries.ts`.** Reading another user's data is the entire point — the gate at the page boundary is what makes this safe. Don't reach into `adminDb.collection("users")...` directly from admin pages; add a helper. The ESLint admin-import gate still applies (admin-queries.ts lives in `lib/firebase/`, the one allow-listed location).
- **Email-prefix search queries `emailLower` (denormalized lowercase), not `email`, with U+F8FF sentinel as `""`.** Firebase Auth doesn't always lowercase emails (Google sign-in can pass mixed case); verbatim-`email` query would silently miss. `emailLower` written alongside `email` by `ensureUserDoc`; legacy users backfilled. The escape-sequence form is required: literal U+F8FF char renders invisible in many editors and silently collapses to empty string under copy-paste, which would turn upper bound into `emailLower < prefix` (always empty).
- **`getAdminCounts` uses three `count()` aggregations, not a full collection scan.** O(N) → 3 reads regardless of size. Tradeoff: cross-snapshot inconsistency (user signing up between reads could be double-counted). Dashboard tolerates ±1 drift in metric tiles. `Math.max(0, ...)` guards so math never goes negative.
- **Admin pages are read-only with ONE controlled exception: the plan-mutation form on `/admin/users/[uid]`.** Every other surface stays read-only. Plan flip goes through `setUserPlanAction` → `adminSetUserPlan` which writes ONLY `plan` + `subscriptionStatus`, deliberately doesn't stamp `lastWebhookAt` (so future real Stripe webhook still passes `isStaleWebhook`). Stripe-owned fields NOT touched. Other admin mutations (comp credits, ban, delete) remain out of scope.
- **Admin layout does NOT share chrome with `(app)`.** Separate header, separate navigation. Prevents accidentally rendering admin chrome to a normal user via a routing bug. Visual cue for admins they're in a privileged surface.
- **Only discovery affordance for `/admin` is a conditional section on `/settings`.** `isAdminEmail(user.email)` gates rendering an "Open admin" link. Header / mobile tab bar deliberately do NOT surface `/admin` — even to admins — so non-admins never see hints. New admin discovery affordances must follow the same conditional pattern; never render an admin link unconditionally.
- **`/admin` is in middleware's PROTECTED_PREFIXES.** Middleware does fast-path cookie-presence check; `requireAdmin()` in the layout is the real verifier. Defense-in-depth pattern.
- **Failed-analyses uses a collectionGroup query.** Requires matching `analyses` composite-collection-group index in `firestore.indexes.json` + `REQUIRED_INDEXES` row. Deploy index BEFORE code lands in prod, or page returns FAILED_PRECONDITION on every render.
- **Stripe subscription history is best-effort.** If Stripe API call fails (network blip, customer not found), page renders empty section instead of crashing. Admin is a triage surface — partial data is more useful than 500.
