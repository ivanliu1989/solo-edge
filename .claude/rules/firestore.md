# Firestore + Repos Rules

Firestore is the source of truth for everything except auth identity. The contract between server, Cloud Function, and dashboard is the document shape — not raw query strings.

- **Document shapes live in `lib/firebase/types.ts`.** Every Firestore read/write uses these types. Changing a shape = update every consumer in the same change. No schema migration tool.
- **Collection paths centralized in `lib/firebase/repos.ts`.** Use the named helpers (`usersCol`, `analysisDoc(uid, id)`, etc.). Never hand-build `db.collection("users")` paths in route handlers.
- **All writes via Admin SDK from server code.** Security rules deny client writes on all server-owned collections. Admin SDK has no auth context and bypasses rules — that's the pattern, not a workaround.
- **Don't reach into `adminDb` directly from route handlers, Server Components, server actions, or Cloud Function handlers.** Add a typed helper to `repos.ts`. Applies to reads AND writes (batched reads like `getAll` belong in repos too). **Enforced** by ESLint: `@/lib/firebase/admin` import is restricted outside `lib/firebase/`.
- **For multi-field reads off `users/{uid}`, use `getUserDocOrNull(uid)`** rather than inlining `userDoc(uid).get() + snap.exists ? (snap.data() as UserDoc) : null`. Use a purpose-specific helper (`getUserSubscriptionStatus`) when you only need ONE field — documents the read intent.
- **Transcripts/large fields are server-only on read too.** They can be large; the dashboard never surfaces them. If a feature needs them in the browser, write a thin Server Component that selects only the fields needed.
- **Use deterministic doc IDs when the input is deterministic.** Example: `analysisDocId(youtubeVideoId, promptVersion) = ${videoId}__${promptVersion.replace(/[^A-Za-z0-9._-]/g, "_")}`. The id appears in URL paths, so URL-safe ASCII only — never `+` (URL-decodes as space and breaks routing).
- **`createOrGetAnalysis()` treats `status: "failed"` as a cache miss** — resets to `queued`, clears prior error, returns `cached: false` so caller re-triggers pipeline. `done` cached forever; in-flight statuses return `cached: true` to avoid double-triggering.
- **Named Firestore database is `{your-db-id}`, NOT `(default)`.** Lets you have separate dbs for separate apps under one Firebase project. `lib/firebase/admin.ts` and `lib/firebase/client.ts` read the id from env (`FIRESTORE_DATABASE_ID` / `NEXT_PUBLIC_FIRESTORE_DATABASE_ID`). `firebase.json` declares `"database": "{your-db-id}"` for the rules deploy.
- **Use `FieldValue.serverTimestamp()` for `createdAt` / `completedAt`.** Cast via `as unknown as Timestamp` when writing (type system can't see through FieldValue). Reads come back as real `Timestamp`; convert with `.toDate()`.
- **`adminDb.settings({ ignoreUndefinedProperties: true })` is set in `admin.ts`.** Writes silently drop undefined. Use `null` for explicit "no value" — undefined and null are NOT equivalent in Firestore.
- **`ensureUserDoc` is idempotent.** Mirrors Firebase Auth → `users/{uid}` on every session-create. Belt + braces guards against edge cases.
- **Three sanctioned billing write helpers** — see `.claude/rules/billing.md`. Never `userDoc(uid).set()` on billing fields directly.
- **Cross-user shared output cache pattern (if your LLM output is deterministic given inputs):** top-level `{kind}Outputs/{id}` collection, written on terminal status, per-user mirror gets `costCents: 0` on cache hit. Two-level cache check (API boundary + pipeline entry) catches the race where another user populated cache between checks. **Invariant:** no user-specific data may flow into LLM prompts — adding personalization corrupts cache for every other user.
- **Security rules are part of the contract.** When you add a new collection or change an existing one, update `firestore.rules` in the same change. Test in the emulator before deploying.
- **`rateLimits/` collection is server-only** (`allow read, write: if false`). Prevents clients from probing remaining quota or resetting their own counter.
- **`mail/` collection (if you use the Firebase Trigger Email extension)** is server-only. Documents auto-deleted after 30 days via `TTL_EXPIRE_VALUE=30 day` config.
- **Indexes: add composite indexes to `firestore.indexes.json`** when introducing new ordered queries. Multi-field orderings need explicit indexes. **Enforced** by `lib/firebase/index-manifest.test.ts` — bidirectional containment check between a `REQUIRED_INDEXES` manifest (naming each call site) and `firestore.indexes.json`. Catches missing indexes (broken queries in prod) AND dead indexes (cost without consumer).
- **Workflow for new composite query:** add query in repos.ts → add REQUIRED_INDEXES row → add block to firestore.indexes.json → `pnpm test` confirms manifest matches → `firebase deploy --only firestore:indexes` BEFORE merging.
