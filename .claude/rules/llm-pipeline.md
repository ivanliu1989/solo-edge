# LLM Pipeline Rules

Task-routed, version-stamped, cost-audited, single-shot. Safe to re-invoke at the same id (cache check at API boundary, not inside pipeline).

- **All LLM calls go through `runTask()` in `lib/llm/router.ts`.** Direct `client.models.generateContent()` bypasses fallback routing, structured-output validation, and the `llmCalls` audit. **Enforced** by ESLint: `@google/genai` (or your provider SDK) restricted outside `lib/llm/providers/` and `lib/transcribe/`.
- **Prompts are versioned files at `lib/llm/prompts/{task}/v{n}.ts`.** Export: `version`, `system`, `buildUser`, `schema`, `schemaName`, `schemaDescription`. The composite `ANALYSIS_PROMPT_VERSION` lands in the Firestore doc ID. **Editing a shipped prompt requires a new `v{n+1}` file.** Same `n` = silent overwrite of cached output at same ID.
- **Separator is `-` (URL-safe ASCII)** in composite versions, never `+`.
- **Output schemas are Zod.** Provider passes `z.toJSONSchema(schema)` as `responseJsonSchema`; runtime re-parses with original schema for validation.
- **Transient retry via `withGeminiRetry`** in `lib/llm/retry.ts`. Retryable: 429, 500, 503, 504. Defaults: 3 attempts, exp backoff 1s/2s/4s ¬±20% jitter, max 60s. Honors `RetryInfo.retryDelay` from quota errors. Non-retryable: 4xx other than 429, Zod failures, RECITATION blocks. Worst case 180s/call (inside 540s GCF timeout).
- **Task routing** in `taskRouting` map in router.ts. Update when a new model lands ‚Äî single source of truth. No production task uses `fallback` today (adds cost, complicates accounting). Add when actually needed.
- **`runTask` wraps terminal provider error in `LLMTaskError`** carrying `{ task, model, provider, cause }`. Upstream catches tag these on Sentry context so triage answers "which task on which model regressed" without scanning stack frames.
- **`runAnalysis` does NOT throw.** On caught error: call `setAnalysisStatus({ status: "failed", statusError })` and return. Throwing races with Cloud Function retry semantics, could leave doc in weird state. The terminal `statusError` write goes through `toUserStatusError()` in `lib/llm/status-error.ts` so raw provider stacks don't leak to user-visible doc.
- **Pipeline error catches MUST `captureException` to Sentry.** The no-throw rule means failures are caught locally ‚Äî without `captureException(err, { site, ...context })`, they're invisible. Use `site:` tag matching the catch location.
- **Per-tier daily cost cap on the entitlement** (`dailyCostCapCents`). Both gates ‚Äî `/api/analyze` route-layer AND `runAnalysis` in-pipeline ‚Äî read via `getUserEntitlement({plan, subscriptionStatus})`. Customer-facing 429: `"Today's processing limit reached. Resets at local midnight."`
- **Two rate gates layered by `discoveredVia`:** route-layer gate (operator `>=`, fires BEFORE doc creation) for manual paste; in-pipeline gate (operator `>`, fires AFTER doc creation) for subscription-discovered work. The strict-greater in-pipeline catches off-by-one (the just-created doc is already in `getTodayAnalysisCount`'s snapshot).
- **`RunAnalysisArgs.discoveredVia` is required** (not optional). Forces callers to declare which path they're on. Manual path skips in-pipeline gate (route already enforced).
- **Cost recorded per-call** via `recordLlmCall` writing an `llmCalls` row + atomically incrementing parent doc's `costCents` via `FieldValue.increment` in single `db.batch()`. Load-bearing: a pipeline that fails midway still has honest cost reflected.
- **Status enum is closed.** Transitions: `queued ‚Üí fetching_metadata ‚Üí transcribing ‚Üí extracting ‚Üí synthesizing ‚Üí critiquing ‚Üí done|failed`. The `critiquing` stage skipped (no transition; `synthesizing ‚Üí done`) when insights draft is empty.
- **Use `setAnalysisStatus()` for status writes** ‚Äî never inline `analysisDoc(...).set({ status })`. Choke point for instrumentation.
- **Captions-then-audio fallback for YouTube input:** try 3-library chain first; on any failure, pass watch URL to Gemini Files API (`fileData.fileUri`). Gemini fetches server-side, sidesteps bot-detection on Cloud Run IPs. Cache transcript with `source: "youtube_captions"` or `"gemini_audio"`.
- **Per-library captures, not just aggregate `CaptionsUnavailableError`.** Site tags per library: `fetchCaptions.youtubei`, `fetchCaptions.youtube-transcript`, `fetchCaptions.youtube-captions-scraper`. Without granularity, a single Sentry issue masks whether one library regressed industry-wide vs all three failed on a single video.
- **`fetchCaptions.youtubei` THROW path downgraded to `captureMessage("info", ...)`** because youtubei.js routinely 403s from Cloud Run IPs (structural). The "0 segments returned" path stays `captureException` ‚Äî that's a real regression signal.
- **Cross-user output cache at `analysisOutputs/{id}`** when output is deterministic given `(input, promptVersion)`. Server-only collection. Two-level cache check (API boundary `createOrGetAnalysis` + pipeline entry `runAnalysis`). Cache hits write `costCents: 0` to per-user doc. **Invariant:** no user-specific data in prompts.
- **`tradingInsights[].ticker` is uppercase exchange symbol or null.** Defense-in-depth: `normalizeTicker()` applied to every insight after critique/pre-critique selection, before `finalizeAnalysis`. Adding a name‚Üísymbol alias goes in the shared module, not the prompt.
- **`quoteTimestampSeconds` populated POST-LLM, never asked from the model.** `findQuoteTimestamp()` substring-matches insight's `supportingQuote` against transcript segments. Returns null on audio-sourced transcript (empty segments), missing quote, or no-match. UI gracefully degrades.
- **Post-LLM enrichment goes at the same pipeline point** ‚Äî after critique/pre-critique selection, before `finalizeAnalysis`. Not inside a prompt, not after doc is written. Pattern keeps prompts stable across data-cleanup iterations.
